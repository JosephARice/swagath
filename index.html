12 Steps to AI-Driven Wikis on GitHub Pages

  1. Create GitHub Repository: Start by creating a new public repository on GitHub to host your wiki files.
  2. Choose Static Site Generator (SSG): Select an SSG compatible with GitHub Pages (like Jekyll - native support, Hugo, Eleventy, or even plain HTML/JS). This tool will build your HTML pages from source files (like .md files).
  3. Local Development Setup: Clone the repository to your local machine and set up the chosen SSG's development environment according to its documentation.
  4. Develop Basic Wiki Structure: Create your initial wiki pages using Markdown (.md files). Organize them logically with folders and create basic navigation (e.g., a sidebar, index page).
  5. Enable GitHub Pages: Go to your repository's Settings -> Pages. Select the branch (usually main or gh-pages) and folder (usually / or /docs) to deploy from. GitHub Pages will build and host your SSG site.
  6. Select AI Feature & Provider: Decide what AI feature you want (e.g., semantic search, Q&A over documents, summarization). Choose an AI provider (like OpenAI, Google Gemini, Cohere) and identify the relevant API.
  7. Obtain API Key: Sign up for the chosen AI provider's service and get an API key. Crucially, plan how to handle this key securely â€“ exposing it directly in client-side code is risky. (Consider environment variables during build or a serverless function proxy, though the latter adds complexity beyond basic GH Pages).
  8. Design AI Interface: Add UI elements to your wiki's template(s) for the AI interaction (e.g., a search input box, a button to submit a question, an area to display results).
  9. Implement Client-Side JavaScript: Write JavaScript code (in your SSG's assets folder) to:
    • Capture user input from the UI elements.
    • Prepare the data/prompt for the AI API call.
    • Use the fetch API to make requests to the AI provider's endpoint, including your (securely handled) API key.
  10. Process & Display AI Results: Enhance your JavaScript to parse the JSON response from the AI API and dynamically update the wiki page's HTML to display the results (e.g., show search results, display an answer).
  11. Test and Refine: Test the AI integration thoroughly. Debug issues, refine the prompts sent to the AI for better results, and improve the user interface based on testing. Test deployment on GitHub Pages.
  12. Commit, Push, and Iterate: Commit your code (SSG configuration, content, templates, JavaScript) to your repository and push it to the branch configured for GitHub Pages. Your site will rebuild and deploy. Continuously add content and improve the AI features.
